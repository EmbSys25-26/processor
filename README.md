# C Language Lexer & Parser — AST Builder

A **Flex + Bison** based front-end for a subset of the C language. It reads a `.c` source file, tokenises it with the lexer, parses it with the grammar, and outputs a pretty-printed **Abstract Syntax Tree (AST)** to stdout.

---

## Project Structure

```
.
├── Makefile
├── program.c               ← full-featured example input file
├── Lexer/
│   └── lexer.l             ← Flex lexer (tokeniser)
├── Parser/
│   ├── parser.y            ← Bison grammar (parser + AST builder)
│   ├── ASTree.h / ASTree.c ← AST node definitions and tree manipulation
│   ├── ASTPrint.h / ASTPrint.c ← AST pretty-printer
│   └── parser.tab.h / parser.tab.c ← (generated by Bison)
├── Util/
│   ├── NodeTypes.h         ← Enums for operator types, qualifiers, variable types
│   ├── logger.h / logger.c ← Coloured terminal logging utilities
└── test_files/
    └── teste.c             ← Additional test input
```

| Tool  | Minimum version |
|-------|----------------|
| GCC   | any modern      |

| Bison | 3.0+            |

On Debian/Ubuntu:
```bash
sudo apt install gcc flex bison
```

## Building

```bash
make
```

This runs Bison on `parser.y` to generate `parser.tab.c` and `parser.tab.h`, then runs Flex on `lexer.l` to generate `lex.yy.c`, and finally compiles everything into the `compiler` binary.

To clean generated files:
```bash
make clean
```

---

## Running

```bash
./compler <file.c>
```

Example:
```bash
./compiler program.c
```

On a successful parse, the tool prints:
```
Parse succeeded. AST:
<tree output>
```

On failure it prints the line number and the token where the error occurred.

## Testing the Lexer Alone

A standalone lexer test binary can be built separately (see `test_lexer.c`). It tokenises the file and prints a table of `LINE | TOKEN | LEXEME` without invoking the parser.

## What C Is Supported

The sections below describe exactly what you can write in a `.c` file that this tool will accept.

---

### Pre-processor Directives

```c
#define MAX 10
```

Preprocessor directives are currently outside the supported subset-C surface. The lexer/parser reject them explicitly rather than building AST nodes for them.

---

### Primitive Types

| Type keyword   | Meaning                        |
|---------------|-------------------------------|
| `int`          | integer                        |
| `char`         | character                      |
| `float`        | single-precision float         |
| `double`       | double-precision float         |
| `long`         | long integer                   |
| `long int`     | same as `long`                 |
| `short`        | short integer                  |
| `short int`    | same as `short`                |
| `long double`  | extended precision float       |
| `void`         | no type / no return value      |

---

### Type Qualifiers

These can be combined with any type in declarations:

| Qualifier    | Effect                                    |
|-------------|------------------------------------------|
| `const`      | marks the variable as non-modifiable      |
| `volatile`   | prevents compiler optimisation on variable|
| `signed`     | explicitly signed integer                 |
| `unsigned`   | unsigned integer                          |

Examples:
```c
const int LIMIT = 100;
volatile int sensor = 0;
unsigned int flags = 0;
signed int delta = -5;
```

---

### Visibility / Storage Qualifiers

| Qualifier  | Effect                              |
|-----------|-------------------------------------|
| `static`   | file-scope or function-local static |
| `extern`   | declares an external variable       |
| `inline`   | hints for inline function expansion |

Examples:
```c
static int counter = 0;
extern int shared;
inline int add(int a, int b) { return a + b; }
```

---

### Pointers

Single and multiple levels of indirection are supported:

```c
int* p;           // pointer to int
int** pp;         // pointer to pointer to int
*p = 10;          // dereference (write)
int x = *p;       // dereference (read)
int* q = &x;      // address-of
```

Pointer types are also valid in casts and function parameters.

---

### Arrays

One-dimensional and multi-dimensional arrays are supported:

```c
int arr[10];          // declaration with fixed size
int matrix[5][5];     // two-dimensional array
arr[0] = 1;           // indexed write
int v = matrix[1][2]; // indexed read
```

Arrays with no size (as function parameters) are also accepted:
```c
float average(float arr[], int n);
```

---

### Structs

```c
struct Point {
    int x;
    int y;
    struct Point* next;   // self-referential pointer member
};

struct Point p;
p.x = 3;                  // member access via dot
p.y = 4;

struct Point* ptr = &p;
ptr->x = 10;              // member access via arrow
```

---

### Unions

```c
union Data {
    int i;
    float f;
};

union Data d;
d.i = 42;
```

---

### Enums

```c
enum Color { RED, GREEN = 5, BLUE };  // members can have explicit values
enum Color c = GREEN;                 // variable of enum type
```

Enum members with negative values are also supported:
```c
enum Dir { FORWARD = 1, BACKWARD = -1 };
```

---

### Variable Declarations

Simple declarations, with or without initialisation:

```c
int x;
int x = 5;
float y = 3.14;
char c = 'a';
int a, b, c;           // multiple variables on one line
int n1 = 1, n2 = 2;   // multiple with initialisation
```

---

### Literals

| Kind              | Examples                                     |
|------------------|---------------------------------------------|
| Integer decimal   | `0`, `42`, `100`                             |
| Integer hex       | `0xFF`, `0x1A2B`                             |
| Float             | `3.14`, `.5`, `1e10`, `2.5e-3`               |
| Character         | `'a'`, `'\n'`, `'\t'`, `'\\'`, `'\0'`        |
| String            | `"hello"`, `"line1\nline2"`                  |

### Operators

#### Arithmetic

| Operator | Meaning        |
|---------|----------------|
| `+`      | addition       |
| `-`      | subtraction    |
| `*`      | multiplication |
| `/`      | division       |
| `%`      | modulo         |

#### Comparison

| Operator | Meaning                  |
|---------|--------------------------|
| `==`     | equal                    |
| `!=`     | not equal                |
| `<`      | less than                |
| `>`      | greater than             |
| `<=`     | less than or equal       |
| `>=`     | greater than or equal    |

#### Logical

| Operator | Meaning     |
|---------|-------------|
| `&&`     | logical AND |
| `\|\|`   | logical OR  |
| `!`      | logical NOT |

#### Bitwise

| Operator | Meaning         |
|---------|-----------------|
| `&`      | bitwise AND     |
| `\|`     | bitwise OR      |
| `^`      | bitwise XOR     |
| `~`      | bitwise NOT     |
| `<<`     | left shift      |
| `>>`     | right shift     |

#### Assignment

```c
x = 5;     // simple
x += 1;    x -= 1;    x *= 2;    x /= 2;    x %= 3;
x &= 0xFF; x |= 0x01; x ^= 0x0F;
x <<= 1;   x >>= 1;
```

#### Increment / Decrement

```c
x++;   // post-increment (higher precedence — binds tighter)
x--;   // post-decrement
++x;   // pre-increment
--x;   // pre-decrement
```

#### Ternary

```c
int max = a > b ? a : b;
```

#### Unary Minus

```c
int neg = -x;
int neg2 = -(x + 1);
```

#### sizeof

```c
sizeof(int)
sizeof(a)
sizeof(struct Point)
```

#### Type Cast

```c
float f = (float) x;
int i   = (int) f;
int* p  = (int*) ptr;
```

### Operator Precedence (highest to lowest)

| Level | Operators                            | Associativity |
|-------|--------------------------------------|---------------|
| 1     | `()` `[]` `.` `->` `++` `--` (postfix) | left          |
| 2     | unary `-` `!` `~` `++` `--` (prefix) `sizeof` cast | right |
| 3     | `*` `/` `%`                          | left          |
| 4     | `+` `-` `<<` `>>`                    | left          |
| 5     | `<` `>` `<=` `>=`                    | left          |
| 6     | `==` `!=`                            | left          |
| 7     | `&`                                  | left          |
| 8     | `^`                                  | left          |
| 9     | `\|`                                 | left          |
| 10    | `&&`                                 | left          |
| 11    | `\|\|`                               | left          |
| 12    | `?:`                                 | right         |
| 13    | `=` `+=` `-=` `*=` `/=` `%=` `&=` `\|=` `^=` `<<=` `>>=` | right |

### Control Flow

#### if / else if / else

```c
if (x < 0) {
    return -1;
} else if (x == 0) {
    return 0;
} else {
    return 1;
}
```

#### switch / case / default

```c
switch (op) {
    case 0:  r = a + b; break;
    case 1:  r = a - b; break;
    default: r = 0;     break;
}
```

Case labels can be integer literals, character literals, or enum identifiers.

#### while

```c
while (n > 0) {
    n--;
}
```

#### do-while

```c
do {
    x++;
} while (x < 10);
```

#### for

```c
for (int i = 0; i < n; i++) {
    // body
}
```

The init field may be empty, a plain assignment, or a variable declaration with assignment. The condition and update fields may also be empty.

#### break / continue

```c
for (...) {
    if (i == 5) continue;
    if (i == 8) break;
}
```

#### return

```c
return;          // void function
return a + b;    // with expression
```

---

### Functions

#### Prototype

```c
int sum(int a, int b);
```

#### Definition

```c
int sum(int a, int b) {
    return a + b;
}
```

#### Calling

```c
int result = sum(x, y);
sum(x + 1, y * 2);      // expressions as arguments
```

#### Variadic parameters

```c
void log(const char* fmt, ...);
```

#### Unnamed parameters

```c
void f(int);   // parameter without a name
```

---

### Comments

Both styles are supported and fully ignored by the parser:

```c
// single-line comment

/* multi-line
   comment */
```

## File Descriptions

### `Lexer/lexer.l`
The Flex tokeniser. It defines regular expressions for every token the language uses — keywords, identifiers, numeric literals (decimal, hexadecimal, float), character literals (including escape sequences like `\n`, `\t`, `\\`, `\'`, `\0`), string literals, all operators, and punctuation. Line numbers are tracked for error reporting. Single-line (`//`) and block (`/* */`) comments are silently consumed. Unknown characters produce a `TOKEN_ERROR`. `#define` and `#undef` are currently rejected explicitly as outside the supported subset-C surface.

### `Parser/parser.y`
The Bison grammar. It is organised into the following groups of rules:

- **`translation_unit`** — the C11-aligned top-level rule. It builds an explicit `NODE_TRANSLATION_UNIT` root whose children are the file-scope external declarations.

- **`external_declaration`** — things valid at file scope in the supported subset: `function_definition` or `declaration`.

- **`statement_sequence`** — an ordered list of statements inside a block. Statements are stored as sibling nodes (a flat linked list), not nested.

- **`statement`** — a single statement valid inside a block: selection (`if`/`switch`), iteration (`for`/`while`/`do-while`), jump (`break`/`continue`/`return`), compound block (`{...}`), declaration, or expression statement.

- **`compound_statement`** — a `{ statement_sequence }` block. It becomes a real `NODE_BLOCK`, and semantic analysis uses that structural node as the lexical-scope boundary.

- **`selection_statement`** — delegates to `if_statement` or `switch_statement`.

- **`if_statement`** — `if (expression) stmt` or `if (expression) stmt else stmt`. The dangling-else ambiguity is resolved with the `LOWER_THAN_ELSE` precedence trick, ensuring `else` always binds to the closest `if`.

- **`switch_statement`** — `switch (expression) { case_list [default_clause] }`. Cases still need later work to become full C11 constant-expression cases.

- **`iteration_statement`** — delegates to `while_loop`, `do_while_loop`, or `for_loop`. `NODE_FOR` is treated as its own lexical scope during semantic analysis so declarations in the init clause remain loop-local.

- **`for_loop`** — preserved as a real `NODE_FOR` with four ordered children: init, condition, step, and body. Semantic analysis treats the `NODE_FOR` as the loop scope boundary so declarations in the init clause remain visible through the condition, step, and loop body, matching C11 `6.8.5.3`.

- **`jump_statement`** — `return`, `break`, `continue`. `return` is now a plain jump node; function ownership is inferred structurally during semantic analysis rather than copied from parser-global state.

- **`declaration`** / **`function_definition`** — declarations now flow through C11-style `declaration-specifiers` plus declarators, while function definitions use `declaration_specifiers declarator compound_statement`.

- **Expression ladder (`expression` through `primary_expression`)** — expressions now follow the C11 `6.5` hierarchy: comma, assignment, conditional, logical-or, logical-and, inclusive-or, exclusive-or, bitwise-and, equality, relational, shift, additive, multiplicative, cast, unary, postfix, and primary. This fixes the old precedence bugs where shifts were grouped with addition, comparisons were flattened with bitwise/logical operators, and assignment was not a real expression family.

- **`expression_statement`** — a bare `;` or `expression ;`. Assignment no longer lives in a separate statement-only grammar path; it is parsed as `assignment_expression`, which matches C11.

- **Postfix/unary nodes** — function calls, array access, address-of, dereference, casts, `sizeof`, and member access now use child-based expression trees instead of the older identifier-special-case encodings. That keeps the AST structurally aligned with C11 postfix/unary expression rules and simplifies later semantic checks.

- **Type non-terminals** (`declaration_specifiers`, `type_specifier`, `type_pointer`, `all_type_specifiers`, `type_name`, `type_cast_specifier`) — build the chain of type/qualifier/pointer nodes that describes a declaration, cast target, or function return type.

### `Parser/ASTree.h` / `Parser/ASTree.c`
The AST data structures and manipulation functions.

- **`TreeNode_t`** — a single node in the tree. Fields: `nodeType` (what kind of node), `nodeData` (a union holding either a `long int`, a `double`, or a `char*` string), `nodeVarType` (for semantic use), `p_firstChild` (first child node), `p_sibling` (next sibling), `childNumber` (count of children), `lineNumber`.

- **`NodeCreate`** — allocates and zero-initialises a new node.

- **`NodeAddChild`** — appends a child to a parent by walking the sibling chain to the end.

- **`NodeAddNewChild`** — allocates a new node and immediately attaches it as a child; a convenience wrapper.

- **`NodeAddChildCopy`** — makes a shallow copy of a node (deep-copying string data) and attaches it as a child. Used for parameters to avoid double-free issues.

- **`NodeAppendSibling`** — appends a node to the end of a sibling list without a parent pointer; used when building statement lists.

- **`ParserObject_t`** — the Bison value type (`YYSTYPE`). A union of `TreeNode_t*` and `NodeData_t`, so both tree nodes and raw literal values can be passed through grammar actions.

### `Parser/ASTPrint.h` / `Parser/ASTPrint.c`
Renders the AST as a Unicode tree diagram to stdout (using `├──`, `└──`, `│` box-drawing characters).

- **`ASTPrint`** — entry point. Iterates over top-level siblings (global declarations) and calls the recursive printer for each.

- **`ASTPrintRecursive`** — depth-first traversal. Prints each node with its type name and, where applicable, its value (identifier name, numeric value, operator symbol, type name, qualifier, etc.).

- **`NodeTypeToStr`** — maps every `NodeType_t` enum value to a human-readable string.

- **`NodeValueSuffix`** — produces the parenthesised annotation shown next to a node (e.g., `IDENTIFIER (x)`, `INTEGER (42)`, `OPERATOR (+)`).

### `Util/NodeTypes.h`
Enumerations that describe the *contents* of a node (as opposed to its structural type):

- **`OperatorType_t`** — every operator the language supports (`OP_PLUS`, `OP_ASSIGN`, `OP_SIZEOF`, etc.), stored in `nodeData.dVal` for `NODE_OPERATOR` nodes.

- **`SignQualifier_t`** — `SIGN_SIGNED` / `SIGN_UNSIGNED`, stored in `NODE_SIGN` nodes.

- **`ModQualifier_t`** — `MOD_CONST` / `MOD_VOLATILE`, stored in `NODE_MODIFIER` nodes.

- **`VisQualifier_t`** — `VIS_STATIC` / `VIS_EXTERN` / `VIS_INLINE`, stored in `NODE_VISIBILITY` nodes.

### `Util/logger.h` / `Util/logger.c`
Coloured terminal logging macros (`LOG_DEBUG`, `LOG_WARNING`, `LOG_ERROR`, `LOG_MESSAGE`) using ANSI escape codes. Short variants (`LOG_DEBUG_SHORT`, etc.) omit the function-name header and can optionally write to `log.txt`. The active log level is controlled by `LOG_LEVEL_SHORT` in `logger.h`.

### `Makefile`
Drives the full build in the correct dependency order:

1. Bison processes `Parser/parser.y` → `parser.tab.c` + `parser.tab.h`
2. Flex processes `Lexer/lexer.l` (which `#include`s `parser.tab.h` for token codes) → `lex.yy.c`
3. GCC compiles and links all `.c` files into the `parser` binary.

`make clean` removes all generated files.

## AST Node Types Reference

| Node type              | What it represents                              |
|-----------------------|------------------------------------------------|
| `NODE_PROGRAM`         | root of the tree                                |
| `NODE_FUNCTION`        | function definition or prototype                |
| `NODE_FUNCTION_CALL`   | a call expression `f(args)`                     |
| `NODE_PARAMETER`       | a parameter in a function signature             |
| `NODE_VAR_DECLARATION` | a variable declaration (`int x = 5`)            |
| `NODE_ARRAY_DECLARATION`| an array declaration (`int arr[10]`)           |
| `NODE_ARRAY_ACCESS`    | an array index expression (`arr[i]`)            |
| `NODE_STRUCT_DECLARATION` | a `struct` type definition                  |
| `NODE_UNION_DECLARATION`  | a `union` type definition                   |
| `NODE_ENUM_DECLARATION`   | an `enum` type definition                   |
| `NODE_STRUCT_MEMBER`   | a field inside a struct or union                |
| `NODE_ENUM_MEMBER`     | a member of an enum (`RED`, `GREEN = 5`)        |
| `NODE_TYPE`            | a type keyword (`int`, `float`, `struct X`, …)  |
| `NODE_SIGN`            | `signed` / `unsigned` qualifier                 |
| `NODE_MODIFIER`        | `const` / `volatile` qualifier                  |
| `NODE_VISIBILITY`      | `static` / `extern` / `inline` qualifier        |
| `NODE_POINTER`         | pointer type in a declaration (`int*`)          |
| `NODE_POINTER_CONTENT` | pointer dereference in an expression (`*p`)     |
| `NODE_REFERENCE`       | address-of expression (`&x`)                    |
| `NODE_TYPE_CAST`       | explicit type cast `(int) x`                    |
| `NODE_OPERATOR`        | any binary or unary operator                    |
| `NODE_TERNARY`         | the `? :` conditional operator                  |
| `NODE_IF`              | `if` statement (2 or 3 children: cond, then, [else]) |
| `NODE_SWITCH`          | `switch` statement                              |
| `NODE_CASE`            | a `case` label and its body                     |
| `NODE_DEFAULT`         | the `default` clause                            |
| `NODE_WHILE`           | `while` loop (also used internally for `for`)   |
| `NODE_DO_WHILE`        | `do { } while` loop                             |
| `NODE_RETURN`          | `return` statement                              |
| `NODE_BREAK`           | `break` statement                               |
| `NODE_CONTINUE`        | `continue` statement                            |
| `NODE_IDENTIFIER`      | a variable or function name                     |
| `NODE_INTEGER`         | an integer literal                              |
| `NODE_FLOAT`           | a float literal                                 |
| `NODE_CHAR`            | a character literal                             |
| `NODE_STRING`          | a string literal                                |
| `NODE_POST_INC`        | postfix `x++`                                   |
| `NODE_POST_DEC`        | postfix `x--`                                   |
| `NODE_PRE_INC`         | prefix `++x`                                    |
| `NODE_PRE_DEC`         | prefix `--x`                                    |
| `NODE_NULL`            | empty / placeholder (e.g. empty block)          |

## Known Limitations

- Preprocessor directives are outside the supported subset-C surface and are rejected.
- `goto` and labels are not supported by the parser (the lexer does not emit a `TOKEN_GOTO`).
- Semantic execution now runs two passes. Implemented checks currently include `SEM001`, `SEM002`, `SEM003`, `SEM004`, `SEM005`, `SEM011`, `SEM040`, `SEM041`, `SEM043`, `SEM050`, `SEM051`, `SEM060`, `SEM061`, and `SEM062`. The full semantic matrix is still incomplete.
- String concatenation of adjacent string literals (`"a" "b"`) is not supported.
